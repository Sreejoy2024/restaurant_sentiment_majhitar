## Overview

This project performs **end-to-end sentiment analysis** on restaurant reviews collected from Majhitar, Sikkim — a small town on National Highway 10 in East Sikkim, India. It implements:

- **Rule-based NLP**: VADER-style lexicon scoring + TextBlob-style polarity analysis (both implemented from scratch — no external NLP packages needed)
- **Machine Learning classifiers**: Logistic Regression, Naive Bayes, Linear SVM with TF-IDF features
- **Ensemble voting** across all models
- **Smart restaurant recommendation** from natural-language food preference queries
- **Interactive Streamlit web app** for live analysis

## Models & Architecture

### Text Preprocessing Pipeline (`src/preprocessing.py`)
```
Raw Review Text
    │
    ├── Lowercase
    ├── Remove URLs, emails, special chars
    ├── Expand contractions (won't → will not)
    ├── Remove stopwords
    └── Feature engineering:
          word_count, char_count, avg_word_length
          exclamation_count, question_count
          positive_word_count, negative_word_count
          sentiment_word_ratio, has_negation
```

### Sentiment Analysis Models (`src/sentiment_analyzer.py`)

#### 1. VADER (Rule-based)
- Lexicon-based scoring with ~80 food-domain words
- Handles **negation** (`not delicious` → negative)
- Handles **boosters** (`very amazing` → stronger positive)
- Compound score normalized to [-1, 1]
- Label: positive (≥0.05), negative (≤−0.05), neutral otherwise

#### 2. TextBlob (Pattern-based)
- Polarity score (−1 to +1) from subjectivity lexicon
- Subjectivity score (0 to 1)
- Negation-aware

#### 3. Logistic Regression (ML)
- TF-IDF vectorizer: 5000 features, unigrams + bigrams
- L2 regularization (C=1.0), LBFGS solver
- 5-fold cross-validation

#### 4. Naive Bayes (ML)
- Multinomial NB with Laplace smoothing (α=1.0)
- Fast and interpretable baseline

#### 5. Linear SVM (ML)
- LinearSVC, C=1.0, max_iter=1000
- Typically best performer on text classification

#### 6. Ensemble (Voting)
- Majority vote: VADER + TextBlob + best ML model
- Robust to individual model weaknesses

## Restaurant Recommender

**How it works:**

```
User Query: "I want something cheesy and dessert type food"
         │
         ├── 1. Extract preference keywords → tags: [cheesy, desserts, cafe, fine_dining]
         ├── 2. TF-IDF cosine similarity against restaurant profiles (40%)
         ├── 3. Tag overlap score with restaurant metadata (40%)
         └── 4. Sentiment boost from review positivity (20%)
                      │
                      ▼
         Ranked restaurant suggestions with explanations
```

**Example Queries:**
```
"I want something cheesy and dessert type food"
→ #1 Frangipani Restaurant (Pal Grove Suites) — cheesy, desserts, fine_dining, scenic_view
→ #2 Rangpo River Café — cafe, desserts, cheesy, river_view

"Looking for spicy north Indian food, budget friendly"
→ #1 Punjabi Kitchen — north_indian, spicy, budget
→ #2 Majhitar Dhaba — budget, cheap, breakfast

"Romantic scenic dinner place"
→ #1 Rangpo River Café — romantic, river_view, scenic
→ #2 Frangipani Restaurant — scenic_view, fine_dining

"Bar with grilled food for late night"
→ #1 Grill and Chill Bar and Restaurant — bar, grilled, late_night, non_veg
```
